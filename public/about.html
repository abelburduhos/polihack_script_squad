How the system works

Two kinds of monitoring zones are supported: forest (wildfire precursors) and flood (water bodies). You define a zone on the map; the system fills it with a hex grid of sensors that stream live readings to a PostgreSQL store.

Place → fill → stream

  1. Pick a sensor type and radius in the control panel, then click anywhere on the map. The zone is persisted with its center, radius, and type.
  2. The server computes a hex grid that fully covers the zone. Each cell gets a sensor with a unique ID, GPS coordinates, and a fixed detection radius (set per type in config.js).
  3. Each sensor emits a periodic reading. Forest sensors track CO₂, temperature, humidity, and a smoke index; flood sensors track water level, flow rate, temperature, and turbidity. Hover any sensor on the map to see the latest reading.
  4. Every reading updates the in-memory state and is persisted to a history table. Delete a zone and all its sensors disappear from both the map and the database.

Configuration

Per-type sensor radius, reading interval, battery drain, and zone bounds live in server/config.js. Edit and restart the server.

Database

PostgreSQL 16 stores zones, sensors (with latest metrics as JSONB), an append-only sensor_readings history table, and an events activity log. Schema in server/schema.sql.
